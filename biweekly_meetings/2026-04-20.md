### Narwhals as a dependency

Polars has deprecated support for the dataframe interchange protocol, which
means scikit-learn now needs library-specific code paths to keep Polars
DataFrames working. The consensus is that adopting
[Narwhals](https://github.com/scikit-learn/scikit-learn/issues/31049) is the
right direction, since the interchange protocol is effectively no longer
implemented by the major dataframe libraries. The existing
[PR to add narwhals as a dependency](https://github.com/scikit-learn/scikit-learn/pull/31127)
is a natural starting point. An open design question remains on whether to
make Narwhals a strict or optional dependency; vendorizing was considered but
would bring a maintenance cost the team would rather avoid. Input on this
trade-off is welcome.

Calibration tools are getting a dedicated push. Antoine Baker will be leading
the work, with a planned call with Olivier Grisel to review existing issues,
discuss the relevant literature, and open new issues as needed. Scope:

- Add an Expected Calibration Error (ECE) metric
  ([#18268](https://github.com/scikit-learn/scikit-learn/issues/18268),
  [#28831](https://github.com/scikit-learn/scikit-learn/pull/28831)).
- Update `calibration_curve` to optionally use quantile bins and support an
  automated choice of bin count
  ([#33658](https://github.com/scikit-learn/scikit-learn/issues/33658)).
- Update the public computation function and the display class accordingly.

There are strong existing opinions in the literature and prior discussions
(e.g. isotonic-regression-based bin definitions) that this work will need to
reconcile.
