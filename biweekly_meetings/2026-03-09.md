### Callback API — sprint toward vote

The callback API ([#33322](https://github.com/scikit-learn/scikit-learn/pull/33322))
design was revisited to add a method call at the beginning of a task (useful for
profiler-like functionality), which means the protocol itself is back under active
design. The team agreed that the protocol must be finalized to support two types of
callbacks: progress bars and metric monitoring
([#33407](https://github.com/scikit-learn/scikit-learn/pull/33407)). Existing hook
names (`on_fit_start` / `on_fit_end`) were found to be confusing and are being
reworked to decouple initialization/tear-down from explicit hook names. The last
possible date to call for a vote for the next release is the end of next week.
An intense review sprint is under way, with dedicated review calls scheduled this
week. Reviews are welcome.

The idea is to call for a SLEP vote early next week!
