## Current focus areas

### Callbacks API

The callbacks API implementation slot is nearing completion. The
[`ScoringMonitor` callback](https://github.com/scikit-learn/scikit-learn/pull/33739)
is almost ready, and [callback support has been added to
`BaseSearchCV`](https://github.com/scikit-learn/scikit-learn/pull/33533). For
the current release, the team decided not to handle metadata routing for
callbacks — meaning the `ScoringMonitor` will initially be limited to scoring
on train data. The broader [Callbacks API
PR](https://github.com/scikit-learn/scikit-learn/pull/33322) continues to
progress.

Reviews welcome on:

### New default colormap: Petroff 10

The team has settled on [Petroff 10](https://github.com/scikit-learn/scikit-learn/pull/33709)
as the new default colormap for scikit-learn visualizations (replacing
`tab10`). The palette is colorblind-friendly and supports up to 10 classes. A
PR is open for review, and once merged, color consistency across all examples
will be updated progressively.

- [#33709](https://github.com/scikit-learn/scikit-learn/pull/33709) — Change default colors in `DecisionBoundaryDisplay` (review welcome)

### Performance on high-core-count CPUs

Work continues on a performance regression affecting gradient boosted trees
and grid searches on machines with high CPU core counts, caused by OpenMP
thread management issues. The plan is to write a reproducer that demonstrates
the impact and report it to OpenMP developers, as well as investigate how to
detect performance cores on macOS and Linux. This work is currently stalled
and input from contributors with HPC/multi-core expertise is welcome.

- [#30662](https://github.com/scikit-learn/scikit-learn/issues/30662) — HistGradientBoosting slowdown related to OpenMP threading
